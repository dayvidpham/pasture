## 1. Verdict: WAIT-bound, not CPU-bound

| Package | real (wall) | user+sys (CPU) | real/CPU | CPU % of wall |
|---------|-------------|----------------|----------|---------------|
| internal/engine | 94.6s | 2.27s | ~42√ó | 2.4% |
| internal/handlers | 103.0s | 2.9s | ~36√ó | 2.8% |
| internal/tasks | 59.6s | 2.4s | ~25√ó | 4.0% |
| cmd/pasture | 100.3s | 4.1s | ~25√ó | 4.1% |
| internal/audit | 17.9s | 3.04s | ~5.9√ó | 17% |

96‚Äì98% of wall time is spent **blocked** (SQLite fsync, DBOS queue-poll, signal timeouts, engine teardown). `-race` is a ~5√ó *CPU* multiplier; multiplying a 2‚Äì4% CPU sliver is invisible against the idle 96‚Äì98% ‚Äî hence `-race ‚âà no-race`. Corroborated by per-test durations clustering on named timeout constants (1s/2s/5s/3s migration floor).

## 2. Ranked root causes (biggest measured wall-clock first)

- **R1 ‚Äî SQLite schema-init/migration tax per fresh DB (~100s+ aggregate, #1).** Every `t.TempDir()` DB pays audit v1‚Üív5 migration + provenance schema. Worst: provenance `seedMLModels` runs **4143 un-batched `INSERT OR IGNORE`** (~1.4s/cold DB). `provenance .../internal/sqlite/db.go:258`; `internal/audit/migrate.go:114-117`. Wasteful, not innate.
- **R5 ‚Äî Zero `t.Parallel()` in engine/handlers/audit (biggest *recoverable* lever).** engine (45 tests), handlers (73), audit (54) parallelize 0%; wall ‚âà sum-of-per-test. Idle-blocked tests overlap at near-zero CPU cost.
- **R2 ‚Äî Per-test engine setup/teardown (~9.8s, ~7s recoverable).** 14 `engine.New` sites √ó ~700ms init. (Shutdown(5s/10s) are *ceilings* ‚Äî normal return 1‚Äì5ms.) Poorly amortized: `boundary_test.go:78` = 1 engine/1 test vs `status_test.go:56` = 21 tests/1 engine.
- **R3 ‚Äî DBOS queue base poll = 1s (~7s in backpressure test).** `dbos-transact-golang@v0.16.0/dbos/queue.go:17`. Legitimate signal-wait; cadence tunable per-queue.
- **R4 ‚Äî Deliberate signal-wait timeouts (1s/2s) exercised honestly.** `slice.go:144` (1s complete), `slice.go:65` (2s start). **Correct pattern already** (bounded timeout on a condition) ‚Äî not offenders.
- **R6 ‚Äî THE one true Sleep offender: `time.NewTimer(5*time.Second)`** at `internal/audit/migrate_v3_backfill_test.go:617` (Scenario12). Fixed race window. Unprincipled.
- **R7 ‚Äî Subprocess `go build` rebuilds inside `t.TempDir()`** (defeats build cache). `migrate_v3_backfill_test.go:342`.
- **Poll-busy-waits (`Now().Add(15s)+Sleep(15ms)`)** at `controller_test.go:42-51`, `control_test.go:84-93`, `subworkflows_test.go` (8+). Wrong primitive; cheap today (exit in 1‚Äì2 iters) but fail slowly (15s) on real hang.

**NON-offenders (cleared):** `controlPollInterval=30s` (control.go:21) and `reviewVotePollInterval=1h` (review.go:59) do **NOT** inflate wall time ‚Äî `ctx.Done()` wins the Recv `select` before `time.After`; sub-ms cancellation. Idle-cadence only.

## 3. Recommendations (no code)

### Quick wins
- **Q1 ‚Äî Batch/skip `seedMLModels`** (one txn, or COUNT-gate). ~1.4s‚Üí~5ms per DB ‚âà **45‚Äì60s aggregate**. *In the provenance dependency ‚Üí needs upstream bump.* Low risk.
- **Q2 ‚Äî Replace the `time.NewTimer(5s)` race window with a Monitor/condition-wait** (readiness signal + bounded timeout). ~5s. Medium risk.
- **Q3 ‚Äî Convert fixed-deadline poll-busy-wait loops ‚Üí Monitor/signal-wait-with-timeout** (projection writer notifies a channel/cond). ~ms wall, but removes the unprincipled pattern and converts slow-hangs to fast-fail.
- **Q4 ‚Äî Cache crash-test build outside `t.TempDir()`** (`sync.Once`, stable path). ~0.46s √ó N.
- **Q5 ‚Äî Lower DBOS queue base polling to ~100ms for test queues** (`WithQueueBasePollingInterval`). ~7s‚Üí~0.7s on backpressure. Low risk (cadence tuning, not sleep removal).

### Structural
- **S1 ‚Äî Golden pre-migrated SQLite DB + skip-migrations, copied per test.** Migrate once in `TestMain`, `CopyFile` per test, open with skip-migrations. ~0.4‚Äì0.6s √ó ~80 tests. Precedent: peasant PR #79 (`storetest.Open`, `golden.go`, `WithSkipMigrations`).
- **S2 ‚Äî Test-sized SQLite pool override via env var** (prod reads env; `TestMain` sets small). Caveat: peasant used pool=**2** not 1 (deadlock when an op holds one conn while requesting another). Precedent: peasant PR #79 `EnvPoolSize` (86.9s‚Üí55.1s cmd).
- **S3 ‚Äî `t.Parallel()` on engine/handlers/audit + per-invocation directory DI (no `t.Setenv`).** *Highest-leverage* (peasant cmd 55.1s‚Üí19.6s on parallelism alone). Estimate handlers ~103s‚Üí~15‚Äì25s. Precedent: peasant PR #79 (221 `t.Parallel()` sites). **See OOM note.**
- **S4 ‚Äî Consolidate engines: share one `engine.New` across many tests** (model `status_test.go:56`). ~7s (~21% suite). Isolate by epoch/workflow id.
- **S5 ‚Äî Hermetic `TestMain` HOME/XDG isolation via `os.Setenv`** (not `t.Setenv` ‚Äî preserves parallelism). Precedent: peasant commit 3d401334.
- **S6 ‚Äî `goleak.VerifyTestMain(m)` per package** ‚Äî surfaces leaked DBOS worker/ticker/signal goroutines before they corrupt parallel `-race` runs. Precedent: peasant 347c0f51.

### OOM-safety (critical before S3)
Peasant added `t.Parallel()` and hit **OOM-SIGTERM** (each test held a prod-sized 2 GiB arena, commit 2eb26eb1). Before locking parallelism: (1) audit every constructor pre-allocating a prod-sized buffer/pool/WAL cache ‚Üí add a `TestMain` env override to shrink it; (2) combine with S2 pool cap so N parallel tests don't open N√óconns; (3) baseline peak RSS via `/proc/<pid>/VmHWM` at `GOMAXPROCS` = CI core count (not `-memprofile` ‚Äî misses off-heap + `-race` shadow). Peasant fit full `-race ./...` in ~26s on 2 vcpu after these. **Do NOT add a nightly flow** ‚Äî race belongs in the normal flow.

## 4. What to do first
Start with **Q1 (batch/skip seedMLModels in provenance)** + **S1 (golden DB + skip-migrations)** ‚Äî they attack R1 (~100s+, the single largest cost) and are mechanical/low-risk. Re-profile, then do **S3 (t.Parallel + dir DI on handlers/engine)** ‚Äî the highest-leverage structural win ‚Äî **but only after** the S3 OOM-safety audit and a `/proc/VmHWM @ GOMAXPROCS=2` baseline. Leave the genuine signal-waits (slice 1s/2s, the 30s/1h Recv cadences) ‚Äî they're already the right pattern. The only Sleep/poll patterns to convert to Monitor/signal-wait are Q2 (5s timer) and Q3 (poll-busy loops).
